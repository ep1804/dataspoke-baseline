# MetaPub: 솔루션 선언문

## 요약

MetaPub은 AI 우선 시대를 살아가는 현대 데이터 조직의 고유한 요구사항을 해결하기 위해 **특별히 설계된 메타데이터 레지스트리 및 데이터 카탈로그**입니다. 모든 사용 사례를 충족하려는 범용 솔루션과 달리, MetaPub은 맞춤화를 핵심 원칙으로 삼아 데이터 생산자, 소비자, AI 에이전트 모두에게 우수한 사용자 경험을 제공합니다.

## 문제: 전문화된 세계 속 범용 솔루션

현대 데이터 조직은 근본적인 불일치에 직면해 있습니다. 기존 데이터 카탈로그 솔루션(DataHub, Open Metadata, Atlas)은 보편적 기능을 추구하지만, 실제 데이터 팀은 범용 도구로는 효율적으로 해결할 수 없는 구체적이고 맥락적인 요구사항을 가지고 있습니다.

### 주요 과제:

1. **차선의 사용자 경험**: 범용 도구는 기능의 깊이보다 폭을 우선시하여, 모든 사람에게는 작동하지만 누구에게도 탁월하지 않은 인터페이스를 제공합니다. 데이터 분석가, 파이프라인 개발자, 도메인 전문가는 모두 전용 최적화가 필요한 고유한 워크플로를 가지고 있습니다.

2. **데이터 메시의 복잡성**: 조직은 여러 팀이 도메인별 파이프라인을 관리하는 데이터 메시 아키텍처를 점점 더 많이 채택하고 있습니다:
   - 중앙 데이터 엔지니어링 팀은 정교한 도구(Flink, Spark)를 사용하여 대규모 비용 효율적 파이프라인을 운영
   - 비즈니스 팀은 더 간단한 주기적 SQL 작업을 실행하거나 수동으로 업데이트되는 테이블을 유지 관리
   - 각 팀은 서로 다른 수준의 복잡성, 가시성, 제어를 요구

3. **AI 전환**: AI 코딩 에이전트의 도입은 두 가지 중요한 방식으로 데이터 인프라 요구사항을 근본적으로 변화시킵니다:
   - **접근 제어 진화**: 개발 및 프로덕션 환경은 AI 에이전트가 라이브 데이터 파이프라인을 손상시키지 않고 안전하게 작업할 수 있도록 재구성되어야 함
   - **검증 인프라**: AI가 생성한 코드와 데이터 변환을 전체 과거 데이터를 스캔하거나 수정마다 이상 탐지 모델을 작성하지 않고도 검증할 수 있는 새로운 시스템이 필요

### 일반화의 역설

데이터 카탈로그 솔루션의 완벽한 일반화는 단순히 어려운 것이 아니라 달성 불가능합니다. 데이터 운영을 성공적으로 확장한 조직은 종종 다양한 이름(고객 데이터 플랫폼, 피처 스토어, 커스텀 레지스트리)으로 자신들의 특정 요구에 맞춤화된 메타데이터 레지스트리 기능을 제공하는 커스텀 솔루션을 구축합니다.

**현실**: 맞춤화가 가능하다면, 맞춤화를 해야 합니다. 현대 AI 지원 개발을 통해 소규모 조직도 이제 이전에는 엄청난 엔지니어링 리소스가 필요했던 맞춤형 솔루션을 구축할 수 있습니다.

## 우리의 솔루션: 목적 있는 맞춤화

MetaPub은 네 가지 맞춤화 기둥 위에 구축되었습니다:

1. **맞춤형 사용자 경험**: 특정 사용자 페르소나와 워크플로를 위한 맞춤 인터페이스
2. **맞춤형 데이터 가용성 기준**: 조직 특화 메트릭 및 임계값
3. **맞춤형 데이터 거버넌스**: 회사의 정책 및 규정 준수 요구사항에 부합
4. **맞춤형 검증 API**: AI 자동화 파이프라인 개발을 위해 특별히 구축된 검증

## 대상 사용자 및 사용 사례

### 데이터 분석가 및 AI 분석 에이전트
**역할**: 데이터 발견 및 탐색

- **스키마 인텔리전스**: 포괄적인 스키마 문서 및 컬럼 의미론
- **계보 추적**: 데이터셋 관계에 대한 명확한 가시성(업스트림 소스 및 다운스트림 의존성)
- **데이터 가용성 메트릭**: 다차원 가용성 뷰(날짜 범위, 제품 유형, 사용자 코호트, 지리적 세그먼트)
- **통계 프로필**: 자동화된 통계 생성
  - 숫자형 변수: 평균, 표준편차, 백분위수, 분포
  - 범주형 컬럼: 카디널리티, 엔트로피, 빈도 분포
- **쿼리 라이브러리**: 엔진 로그에서 추출한 큐레이션된 SQL 샘플 및 자주 사용되는 쿼리 패턴

### 데이터 엔지니어 및 데이터 스튜어드
**역할**: 데이터 품질 보증 및 모니터링

- **가용성 모니터링**:
  - 데이터 도착 빈도 기대치 구성
  - 수량 검사 규칙 및 알림 임계값 설정
  - 사용 통계 및 접근 패턴 추적

- **일관성 및 무결성 모니터링**:
  - 컬럼별 품질 검사 규칙 정의
  - 진화하는 데이터에 대한 시계열 이상 탐지 구현
  - 데이터 검증 경계 설정

### AI 파이프라인 개발 에이전트
**역할**: 자동화된 검증 및 확인

- **파이프라인 검증 프레임워크**:
  - 수정된 파이프라인 출력과 과거 샘플 비교
  - 결과가 설정된 품질 경계를 준수하는지 확인
  - 코드 변경에 대한 자동화된 회귀 탐지

### 정보 보안 팀
**역할**: 규정 준수 및 접근 거버넌스

- **접근 수준 모니터링**: 데이터 접근 권한 추적 및 감사
- **개인정보 관리**: 데이터 개인정보 분류 및 처리 요구사항 모니터링 및 시행

## 설계 원칙

MetaPub은 처음부터 모듈식, 유지보수 가능, AI 친화적으로 설계되었습니다.

### 아키텍처 분리

**프론트엔드, 백엔드, API 레이어의 엄격한 분리**

- **API 레이어**: 최대 유연성과 상호 운용성을 위한 RESTful API 또는 GraphQL
- **독립형 API 문서**: 전용 디렉토리에서 독립적인 명세로 유지 관리
  - **근거 1**: AI 지원 개발 및 검증을 위한 빠른 반복 주기 활성화
  - **근거 2**: 전체 백엔드 구현 없이 API 우선 개발 촉진
  - **근거 3**: 팀 간 협업을 위한 명확한 계약 제공

### 기술 스택

**백엔드: Python 기반 API 서버**

- **주요 근거**: Python 생태계는 다음과 같은 비할 데 없는 기능을 제공합니다:
  - 다양한 데이터 스토리지 시스템에 대한 강력한 커넥터
  - 데이터 검증에 사용할 수 있는 풍부한 통계 및 머신러닝 라이브러리
  - 광범위한 데이터 조작 및 분석 도구
- **부차적 이점**:
  - 강력한 타입 지원(타입 힌트 포함)
  - 성숙한 API 프레임워크(FastAPI, Flask, Django)
  - 광범위한 인재 풀 및 커뮤니티 지원

**프론트엔드: TypeScript 기반 최신 프레임워크**

- **주요 선택**: Next.js 또는 유사한 TypeScript 프레임워크
- **근거**:
  - 강력한 생태계를 갖춘 업계 표준 도구
  - 타입 안전성으로 런타임 오류 감소 및 유지보수성 향상
  - 우수한 개발자 경험 및 AI 코딩 지원
- **시각화**: 전문적인 차트 라이브러리(예: Highcharts, D3.js, Recharts)
  - 풍부하고 인터랙티브한 데이터 시각화
  - 데이터 집약적 애플리케이션을 위한 프로덕션 준비 컴포넌트

## 가치 제안

MetaPub은 범용 만능 데이터 카탈로그에서 목적에 맞게 구축된 조직 맞춤형 메타데이터 인프라로의 패러다임 전환을 나타냅니다:

- **우수한 사용자 경험**: 가상의 워크플로가 아닌 실제 워크플로에 최적화된 인터페이스
- **AI 네이티브 아키텍처**: 처음부터 AI 에이전트를 일급 사용자로 지원하도록 구축
- **유연한 거버넌스**: 조직의 특정 규정 준수 및 정책 요구사항에 맞춤 가능
- **확장 가능한 검증**: AI 기반 개발 속도에 맞춰 확장되는 효율적인 데이터 품질 검사
- **미래 지향적 설계**: 조직의 데이터 성숙도와 함께 진화하는 모듈식 아키텍처
